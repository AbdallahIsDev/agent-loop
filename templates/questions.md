# Questions

Only the most critical, high-stakes questions belong here. This is not for routine clarifications or questions the agent already knows the best answer to.

**Before adding a question, ask yourself:** Do I truly lack the information or context to make a confident decision? If I already know the right approach, just execute it — don't ask.

## When to Add a Question

Only add a question when ALL of these are true:
1. The decision significantly impacts architecture, security, user experience, or project direction
2. You have investigated and still cannot determine the right answer with confidence
3. Making the wrong choice without input would cause meaningful rework or risk

## When NOT to Add a Question

Do NOT add questions for:
- Routine implementation decisions (choose the best practice and execute)
- Minor preferences (naming, formatting, color choices)
- Anything you can determine through investigation, testing, or reasoning
- Questions where either option would work fine
